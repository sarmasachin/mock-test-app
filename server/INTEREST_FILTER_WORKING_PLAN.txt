GOAL
----
1. Login ke baad user apni INTEREST (subcategory / exam topic) choose kare.
2. Default: app me sirf interest-matching tests / categories dikhein.
3. User ke paas hamesha “Saare tests dekho” option ho (toggle ya link).
4. Admin jab test add/edit kare, wahan ek CHECKBOX ho:
   “User ko dikhao” (show in user-facing interest/catalog flows).
   — Alag se implement karenge; is plan me spec locked hai.

INTEREST UNIT (DECISION)
------------------------
Primary: test.subcategory (examCategories level3 ke saath align).
Fallback: test title sirf tab jab subcategory blank ho (discourage).

Published (is_published) ≠ “user ko dikhao”:
  - is_published = test live hai (apply/start cycle chal sakta hai)
  - show_to_users (naya) = interest picker + filtered catalog me list ho

ADMIN CHECKBOX (FUTURE — Phase 1b)
----------------------------------
Location: Admin → Tests → Add/Edit test form (Published checkbox ke paas).

Label (Hindi UI):
  [ ] User ko dikhao (interest / test list me)
English key: showToUsers

DB column (proposed):
  tests.show_to_users BOOLEAN NOT NULL DEFAULT true

Semantics:
  true  = test interest picker + “mere tests” catalog + subcategory list me aa sakta hai
          (subject to is_published + publishAt/validUntil/unpublishAt visibility)
  false = user-facing LISTS me hide; test published reh sakta hai for:
          - users who already applied
          - admin / resolve-by-id (optional Phase 2)

API:
  - mapTest() + admin GET/PATCH include showToUsers
  - GET /v1/tests filters: AND (show_to_users IS NOT FALSE) for public catalog
  - GET /v1/me/interests picker source: same filter

Admin list badge:
  Published + hidden from users → “Published (hidden from catalog)”

FILES (later):
  - server migration
  - server/src/routes/admin.js, tests.js
  - admin-web/src/App.tsx (test form checkbox)
  - app ApiModels + filters in ContentRepository

PHASE 1 — Server: user interests (1 day)
====================================================================

DB
--
user_test_interests (
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  subcategory TEXT NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  PRIMARY KEY (user_id, subcategory)
)

API
---
GET  /v1/me/interests
     → { subcategories: ["HP GK", "SSC CGL"] }

PUT  /v1/me/interests
     body: { subcategories: string[] }
     max 20, trim, dedupe, case-insensitive merge

GET  /v1/tests?subcategory=X          (existing)
GET  /v1/tests?subcategories=a,b,c    (new, optional OR filter ILIKE)

Public catalog query adds (when Phase 1b lands):
  AND COALESCE(show_to_users, true) = true
