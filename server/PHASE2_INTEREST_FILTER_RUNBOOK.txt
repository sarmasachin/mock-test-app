BEHAVIOUR
---------
1. User picks tests on post-login screen.
2. App derives subcategory from each test card (fallback: title).
3. Saves locally + PUT /me/interests (best effort; local wins if offline).
4. On next login / cold start, GET /me/interests restores interests.
5. If server empty but old titles exist locally → migrate titles → subs once.

API (requires Phase 1 server deployed)
--------------------------------------
GET  /v1/me/interests
PUT  /v1/me/interests  { "subcategories": ["HP GK"] }
GET  /v1/tests?subcategories=HP GK,SSC

404 on interests endpoints → local-only mode (old server).
